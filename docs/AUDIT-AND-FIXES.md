## ۲. مشکلات بحرانی پنل مدیریت (Critical Admin Panel Bugs)

### ۲.۱. ویرایش قیمت خدمات بدون ذخیره در دیتابیس ❌ → ✅

**مشکل:** در تب **«نرخنامه»**، دکمه «ذخیره» تخفیف/قیمت فقط state محلی React را تغییر می‌داد و هیچ API فراخوانی نمی‌کرد. با رفرش صفحه، تغییرات از بین می‌رفت.

**راه‌حل:** تابع `savePrice` بازنویسی شد تا با `POST /api/cms` (action=upsert, table=services) قیمت و درصد تخفیف را در دیتابیس ذخیره کند.

### ۲.۲. دکمه‌های ویرایش/حذف اپراتور فقط نمایشی بودند ❌ → ✅

**مشکل:** دکمه‌های ویرایش و حذف در کارت هر اپراتور هیچ handler نداشتند. امکان مدیریت واقعی اپراتورها وجود نداشت.

**راه‌حل:**
- افزودن stateهای `editingOpId` و `opActionMsg`.
- پیاده‌سازی `startEditOperator` برای پر کردن فرم ویرایش.
- پیاده‌سازی `deleteOperator` با `DELETE /api/admin/operators`.
- افزودن متد `DELETE` به API `/api/admin/operators` برای تغییر نقش به `CUSTOMER` و حذف دسترسی‌ها.

### ۲.۳. درخواست برداشت عملیاتی نبود ❌ → ✅

**مشکل:** در تب **«مالی و پرداخت»**، فرم درخواست برداشت فقط نمایش بود و دکمه «ثبت درخواست برداشت» هیچ API فراخوانی نمی‌کرد.

**راه‌حل:**
- پیاده‌سازی `submitWithdrawal` با `POST /api/wallet/withdraw`.
- نمایش پیام موفقیت/خطا پس از ثبت.

### ۲.۴. عدم مدیریت درخواست‌های برداشت توسط مدیر ❌ → ✅

**مشکل:** هیچ UI برای نمایش، تأیید یا رد درخواست‌های برداشت در پنل مدیر وجود نداشت. اگرچه APIهای `/api/admin/withdrawals` و approve/reject پیاده‌سازی شده بودند، اما استفاده نمی‌شدند.

**راه‌حل:**
- افزودن بخش «درخواست‌های برداخت» در تب «مالی و پرداخت» فقط برای `SUPER_ADMIN`.
- نمایش لیست درخواست‌ها با فیلتر وضعیت.
- امکان تأیید با آپلود URL رسید و یادداشت مدیر.
- امکان رد با ثبت دلیل و بازگشت خودکار مبلغ به کیف پول اپراتور.

- استفاده از API موجود `/api/admin/game-config`.
- افزودن متد `DELETE` به API `/api/admin/game-config`.

## ۵. موارد باقی‌مانده برای بهبود آینده (Future Improvements)

> این موارد در بازبینی شناسایی شدند اما به دلیل محدودیت زمانی/تمرکز بر رفع باگ‌های بحرانی، در این نسخه اعمال نشدند. برای نسخه‌های بعدی پیشنهاد می‌شوند:

1. **تقسیم پنل مدیریت به کامپوننت‌های کوچک‌تر:** فایل `admin/page.tsx` هنوز حدود ۱۰۰۰ خط است. بهتر است به `src/components/admin/` تقسیم شود.
2. **افزودن مدیریت کاربران (Users):** مشاهده، مسدودسازی، شارژ دستی کیف پول.
3. **افزودن کدهای تخفیف:** CRUD کامل کدهای درصدی/مبلغی با محدودیت و انقضا.
4. **افزودن گزارش‌ها و BI:** نمودار درآمد، پرفروش‌ترین خدمات، عملکرد اپراتورها.
5. **افزودن Broadcast/اعلان گروهی:** ارسال پیام به کاربران از پنل.
6. **افزودن مدیریت پیش‌تعریف ادمین‌ها (Predefined Admins):** ادمین کردن با آیدی عددی تلگرام/بله قبل از ثبت‌نام.
7. **تقویت Live Edit:** اضافه‌کردن Rich Text Editor، Icon Picker، Color Picker کامل، Batch Save و Preview Mode.
8. **بهبود UI/UX:** یکنواخت‌سازی بیشتر استفاده از `glass`/`glass-strong` به‌جای `bg-white/5` در تمام فرم‌ها و کارت‌ها.
9. **افزودن Dual-View Workspace برای اپراتورها:** نمایش همزمان مدارک و فرم در تب سفارشات.
10. **تقویت چت سفارش:** ارسال فایل، ویس، درخواست OTP.

## ۵. پیکربندی کامل از Environment Variables (Cloudflare)

### ۵.۱. هدف
طبق درخواست کاربر، کل پیکربندی حساس (توکن‌ها، رمزها، درگاه) باید از متغیرهای محیطی Cloudflare خوانده شود و هیچ‌گونه تنظیم دستی در `.env` یا `system_settings` برای راه‌اندازی اولیه لازم نباشد.

### ۵.۲. تغییرات اعمال‌شده
| فایل | تغییر |
|---|---|
| `src/db/seed.ts` | `passwordHash` ادمین/اپراتورها به `null` تنظیم شد تا از `SUPER_ADMIN_PASSWORDS` / `OPERATOR_ADMIN_PASSWORDS` استفاده شود. |
| `src/app/api/auth/admin-login/route.ts` | در صورت `passwordHash=null`، رمز از env خوانده و `needsPasswordChange=true` برمی‌گرداند. |
| `src/app/api/auth/admin-change-password/route.ts` | رمز فعلی پیش‌فرض env به‌عنوان رمز فعلی پذیرفته می‌شود. |
| `src/app/api/admin/operators/route.ts` | اپراتورها با `passwordHash=null` و `mustChangePassword=true` ساخته می‌شوند. |
| `src/lib/bot/engine.ts` | `getBotToken` ابتدا `TOKEN_TELEGRAM` / `TOKEN_BALE` env را می‌خواند. |
| `src/lib/payment/gateway.ts` | `getActiveGateway` و `getGatewayConfig` ابتدا `PAYMENT_GATEWAY` و `ZARINPAL_API` env را می‌خوانند. |
| `src/app/admin/page.tsx` | گیت‌گذاری UI بر اساس مجوزهای دانه‌ای (RBAC) برای تب‌های رسیدها، ربات‌ها، اخبار، بازی‌ها، CMS و تنظیمات. |
| `.env.example` | به‌روزرسانی با نام‌های جدید env: `SUPER_ADMIN_PASSWORDS`, `OPERATOR_ADMIN_PASSWORDS`, `TOKEN_TELEGRAM`, `TOKEN_BALE`, `ZARINPAL_API`, `PAYMENT_GATEWAY`. |
| `docs/CLOUDFLARE-DEPLOYMENT.md` | به‌روزرسانی راهنمای env bindings و چک‌لیست. |
| `docs/ADMIN-GUIDE.md` | اضافه‌شدن بخش RBAC و نحوهٔ ورود env-based. |
| `README.md` | به‌روزرسانی راه‌اندازی محلی و Cloudflare. |

### ۵.۳. مجوزهای دانه‌ای (RBAC) در پنل ادمین
- هر اپراتور دارای آبجکت `permissions` در جدول `users` است.
- SUPER_ADMIN همهٔ مجوزها را دارد.
- مجوزها شامل دسته‌های: `orders`, `services`, `receipts`, `withdrawals`, `payments`, `bots`, `news`, `games`, `cms`, `settings`, `operators` هستند.
- هر دسته دارای `view` و `manage` (و در موارد خاص `approve`/`reject`/`assign`/`changeStatus`/`edit`/`withdraw`) است.
- دکمه‌های مدیریتی در `admin/page.tsx` فقط در صورت داشتن مجوز مربوط نمایش داده می‌شوند.
